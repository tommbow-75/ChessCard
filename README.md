# ChessCard 遊戲專案

> **專案概述**：ChessCard 是一款結合「中國象棋」基礎規則與「殺戮尖塔 (Slay the Spire)」爬塔機制的卡牌策略遊戲。玩家將在棋盤上進行戰鬥，透過吃子削減敵方士氣，並利用獲取的謀略點數搭配卡牌系統來取得勝利。

## Tech Stack (技術與環境)
* **Game Engine:** Godot
* **IDE / Editor:** 推薦使用 Antigravity 進行腳本編輯，或使用 Godot 內建編輯器

## Core Mechanics (核心玩法機制)

### 1. 勝負與士氣系統 (Morale System)
* **初始狀態**：雙方開局各自擁有 **100 點士氣**。
* **扣減機制**：每當一方的棋子被吃掉時，會扣除該方對應的士氣值。
* **勝利條件**：率先將敵方士氣歸 **0** 的一方即獲勝。

### 2. 謀略點數 (SP: Strategy Point)
* **獲取方式**：每當吃掉敵方棋子時，己方會獲得 SP。
* **資源消耗**：SP 是用來發動「卡牌」的專屬資源。

### 3. 回合流程 (Turn Structure)
每回合開始時，玩家自動 **抽 1 張牌**，並且在該回合內擁有 **2 個行動階段**：
1. **行動一 (Board Action)**：移動場上的一枚棋子。
2. **行動二 (Card Action)**：消耗 SP 使用一張卡牌。

## Card System (卡牌系統資料結構)
遊戲中的卡牌分為「召喚卡 (Summon)」與「謀略卡 (Strategy)」兩大類。

### 1. 召喚卡 (Summon Card)
用於在棋盤上召喚實體棋子。
* `Name` (名稱)：卡牌的名稱。
* `Type` (種類)：召喚類（細分為：帥、士、象、車、馬、砲、兵）。
* `Morale_Cost` (士氣值)：該召喚物被吃掉時，己方需扣除的士氣值。
* `Movement` (移動方式)：通常依照象棋原種類規則，但可帶有特殊移動方式。
* `Effect` (特殊效果)：該卡牌進場或存在時的附加效果。
* `SP_Cost` (消耗謀略)：發動此卡牌需要消耗的 SP 點數。

### 2. 謀略卡 (Strategy Card)
用於發動一次性的戰術效果。
* `Name` (名稱)：卡牌的名稱。
* `Type` (種類)：謀略類。
* `Effect` (卡牌效果)：發動後產生的具體遊戲影響。
* `SP_Cost` (消耗謀略)：發動此卡牌需要消耗的 SP 點數。
